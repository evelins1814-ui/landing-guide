> Этот файл — для лендингов, которые **Рекруто делает для собственных вакансий** (не для клиентских). Загружать вместе с общей «Инструкцией по лендингам». ## 1. О компании (использовать как источник правды)

**Название:** Рекруто (`recruto.ru`)

**Что мы делаем:** стартап на стыке HR и IT. Занимаемся аутстаффингом рекрутеров и глубокой оценкой кандидатов через интервью и тестирование. Спрос на услуги подтверждён, активно масштабируемся.

**Год основания:** 2020. На рынке 5+ лет.

**Корпоративная самоидентификация (точные формулировки с hh.ru, использовать дословно):**
- «Подбор персонала — наше ремесло».
- «Учим разговаривать с людьми так, чтобы за 30 минут стало понятно — наш человек или нет».
- «Всю прибыль вкладываем в технологии и в улучшение условий труда».

**Статусы и аккредитации (формулировать аккуратно):**
- Проект реализуется в экосистеме аккредитованной IT-компании и резидента Сколково.
- ⚠️ **Не писать «Рекруто — аккредитованная IT-компания и резидент Сколково» напрямую.** Аккредитация и резидентство относятся к юр.лицу-экосистеме, а не к самому Рекруто. Корректная формулировка: «Проект реализуется в экосистеме...» или «работаем под крылом аккредитованной IT-компании и резидента Сколково». Это важно для соответствия фактам.

**Главные преимущества Рекруто (упоминаются на лендингах ВСЕГДА):**

Это сквозные характеристики компании — то, что делает Рекруто «Рекруто», независимо от конкретной роли. График работы, удалённый или офисный формат, конкретные задачи — у разных вакансий разные. А эти четыре пункта — общие для всех.

1. **Свой отдел разработки «Deepra».**
   У Рекруто есть собственная команда инженеров, которая делает продукты для HR — CRM, планировщик «Слотто», внутренний мессенджер, ИИ-инструменты. Не подписка на чужой софт, а собственная разработка под свои процессы.

2. **Активное внедрение ИИ и автоматизация всех процессов.**
   Рутина уходит машинам — рассылка слотов, бронирование времени, напоминания, ведение базы кандидатов, общение через единый мессенджер. Сотрудник делает только то, что нельзя поручить машине.

3. **Гибкость в процессах — прислушиваемся к сотрудникам.**
   Задачи, график, зоны ответственности всегда обсуждаются. Не «вот должностная инструкция, исполняйте» — а «давайте подумаем, как удобнее именно вам». Это про **культуру**, не про удалёнку. На некоторых вакансиях из этого следует свободный график или удалёнка, на других — нет.

4. **Экосистема IT-аккредитации и резидентства Сколково.**
   Проект развивается в рамках аккредитованной IT-компании и резидента Сколково. Формулировать аккуратно (см. раздел 1) — не «мы аккредитованная компания», а «проект реализуется в экосистеме».

Эти четыре пункта — **сквозные**. Они появляются на любом лендинге Рекруто как фундамент.

**Инструменты-доказательства (используются по релевантности к роли):**

Это конкретика, которая **подкрепляет** главные преимущества. Подбирать только те, что уместны под конкретную вакансию — не вываливать все сразу.

- **Собственная CRM-система** — все вакансии, кандидаты, история общения в одном месте. Сделана внутри Deepra. Подкрепляет автоматизацию.
- **Собственный мессенджер** — рекрутеры не общаются с кандидатами из личных аккаунтов. Связь через Telegram и Max из единого интерфейса. Рабочий номер — компании, а не сотрудника. Сделан внутри Deepra.
- **Планировщик «Слотто»** — внутренний сервис рассылает кандидатам свободные слоты, бронирует время, шлёт напоминания. Сделан внутри Deepra.
- **ИИ-инструменты** — для скрининга, оценки, обработки данных по кандидатам.
- **Экспертиза в рекрутменте** — «учим разговаривать с людьми так, чтобы за 30 минут стало понятно — наш человек или нет».

**Дополнительные преимущества (использовать ТОЛЬКО если они реально есть в конкретной вакансии):**

Эти пункты **не сквозные** — зависят от конкретной роли. Не дописывать «по умолчанию», только если явно сказано в брифе или вакансии.

- Удалённый формат / гибкий график (если есть в этой вакансии)
- Высокий доход для удалённого формата (если применимо)
- Продуманная адаптация
- Возможность карьерного роста

**Цифры компании (брать как факт, можно использовать на любом лендинге):**
- 400 000+ кандидатов обработано
- 1000+ людей вышли на работу
- 9 из 10 клиентов приходят повторно
- год основания: 2020

## 2. Палитра (CSS-переменные — копировать как есть)

```css
:root {
  --blue: #2567fc;          /* основной акцент, кнопки, активные элементы */
  --blue-2: #5199ff;        /* светлый синий для градиентов и hover */
  --blue-deep: #0a1b3d;     /* тёмно-синий для тёмных секций */
  --ink: #0a0c14;           /* основной цвет текста */
  --ink-2: #1c2030;         /* приглушённый текст, второй уровень */
  --muted: #6b7180;         /* серый для подписей, мета-данных */
  --paper: #ffffff;         /* основной фон */
  --paper-2: #f4f6fb;       /* фон карточек, чередующихся секций */
  --line: rgba(10, 12, 20, 0.08);       /* основные разделители */
  --line-soft: rgba(10, 12, 20, 0.05);  /* мягкие разделители */
}
```

**Правила использования:**
- Основной фон — белый (`--paper`), карточки/секции-чередования — `--paper-2`.
- Шапка и анкета — на тёмном фоне (`--ink` или `--blue-deep`), потому что **логотип Рекруто белый** и читается только на тёмном.
- Акцентный синий `--blue` — для кнопок, активных слотов, выделенных цифр, ссылок.
- Не использовать другие яркие цвета (зелёный, оранжевый, фиолетовый). Палитра монохромная: синий + нейтральные.

## 3. Шрифт

**Montserrat** — все веса от 400 до 900.

Подключение:
```html
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link href="https://fonts.googleapis.com/css2?family=Montserrat:wght@400;500;600;700;800;900&display=swap" rel="stylesheet">
```

CSS-переменная:
```css
:root {
  --font: 'Montserrat', system-ui, -apple-system, 'Segoe UI', Roboto, sans-serif;
}
```

**Веса по назначению:**
- 900 — hero-заголовок, главные крупные числа
- 800 — заголовки секций
- 700 — подзаголовки, кнопки
- 600 — выделенные слова внутри текста
- 500 — мета-подписи, навигация
- 400 — основной текст

## 5. Шапка и подвал — фиксированные правила

### Шапка
- Фон: тёмный (`--ink` или `--blue-deep`).
- Слева: логотип Рекруто (data-URI ниже).
- Справа: кнопка-ссылка «Вакансия на hh.ru» с прямой ссылкой на оригинал.
- На мобильном — шапка компактная, без бургер-меню.

### Подвал
- Фон тёмный, в тон шапке.
- Слева: тот же логотип.
- Текст копирайта: `© Рекруто, 2020–[текущий год] · аккредитованная IT-компания · резидент Сколково`
- Никаких email-адресов и личных контактов рекрутёров.

## 6. Логотип

Логотип белый, поэтому требует тёмного фона. Использовать как `<img>` с data-URI ниже:

```html
<img src="data:image/png;base64,iVBORw0KGgoAAAANSUhEUgAAAioAAACKCAQAAAAuuqn..." alt="Рекруто" class="logo">
```

**Полная data-URI лежит в отдельном файле `recruto-logo-datauri.txt`** (приложен к brand kit). Скопировать оттуда строку целиком и вставить в `src`.

Альтернатива — если логотип потерялся, можно собрать текстовый вариант:
```html
<div class="logo-text">РЕКРУТО</div>
```
```css
.logo-text {
  font-family: var(--font);
  font-weight: 900;
  font-size: 24px;
  letter-spacing: -0.02em;
  color: #fff;
}
```

## 7. Корпоративный голос — как мы говорим

### Принципы
- **Сдержанно, по делу, без пафоса.** «Подбор персонала — наше ремесло» — да. «Команда мечты» — нет.
- **Конкретно.** Цифры, факты, инструменты. Не «современный подход», а «собственная CRM и планировщик Слотто».
- **С позиции уверенного профессионала.** Без хвастовства, но и без скромности — мы знаем, что делаем.
- **На «вы», но не канцелярит.** «Работайте, когда и где хочется» — да. «Уважаемый кандидат, мы предлагаем вам...» — нет.

### Любимые формулировки (можно использовать)
- «Подбор персонала — наше ремесло»
- «Работайте, когда и где хочется»
- «Откуда угодно — лишь бы был интернет и спокойное место для разговора»
- «Платим за результат, а не за присутствие»
- «Вы делаете только то, что нельзя поручить машине»
- «Никаких личных аккаунтов»
- «Не нужно согласовывать вручную»

### Запрещённые слова и обороты для лендингов Рекруто
- ❌ «Контора» (даже в негативном контексте — звучит грубо)
- ❌ «Подработка» (обесценивает работу)
- ❌ «Рекрутинг» — мы говорим **«рекрутмент»**
- ❌ «Простая работа», «несложно» (звучит снисходительно)
- ❌ Сравнения «у нас лучше, чем в других местах» — объясняем свои условия, не сравниваем
- ❌ «Серьёзная компания» (звучит как из 2005-го)

### Зацепки для hero-заголовков Рекруто (банк готовых формулировок)
- «Работать, когда и где хочется»
- «Подбор персонала — наше ремесло»
- «Свободы больше, чем кажется»
- «Одна задача: говорить с людьми»

Структура: **4 крупные карточки** с четырьмя главными преимуществами. Не больше, не меньше.

**Карточка 1: Свой отдел разработки Deepra.**
- Заголовок: «Свой отдел разработки — Deepra»
- Подзаголовок (одна строка): «Свои продукты для HR. Не подписка на чужой софт — собственная команда инженеров под наши процессы».

- Заголовок: «Автоматизируем всё, что можно автоматизировать»
- Подзаголовок (одна строка): «Рутина уходит машинам. Сотрудник делает только то, что нельзя поручить ИИ».

**Карточка 3: Гибкость в процессах.**
- Заголовок: «Прислушиваемся к сотрудникам»

⚠️ **Не подменять «гибкость в процессах» на «гибкий график» или «удалёнка».** Это разные вещи. Гибкий график и удалёнка — это конкретные условия конкретной роли, а не сквозное преимущество компании. Свободный график есть не у всех вакансий, а «прислушиваемся к сотрудникам» — есть всегда.

### 8.3. Блок «Инструменты-доказательства» (опциональный, подбирается под акцент)

Этот блок **усиливает** главные преимущества из 8.2, показывая конкретные продукты Deepra. Использовать **не все сразу**, а подбирать релевантные под акцент вакансии.

| Инструмент | Когда вставлять | Содержание карточки |
|---|---|---|
| **Собственная CRM** | Если акцент на автоматизацию или удобство процессов | «Все вакансии, кандидаты, история общения — в одном месте. Сделана внутри Deepra». |
| **Собственный мессенджер** | Если акцент на заботу о сотрудниках, конфиденциальность | «Никаких личных аккаунтов. Telegram и Max из одного интерфейса. Рабочий номер — компании, не ваш». |
| **Планировщик Слотто** | Если у роли есть встречи/интервью с внешними людьми | «Внутренний сервис рассылает свободные слоты и бронирует время. Не нужно согласовывать вручную». |
| **ИИ-инструменты** | Если акцент на технологии, ИИ, обработку данных | «Скрининг, оценка, обработка данных по кандидатам — с помощью наших ИИ-инструментов». |
| **Экспертиза в рекрутменте** | Если акцент на обучение, рост, профессионализм | «Учим разговаривать с людьми так, чтобы за 30 минут стало понятно — наш человек или нет». |

**Правило:** 2–3 карточки в блоке. Не больше. Меньше — лучше, чем больше.

### 8.4. Мокап «Слотто» (CSS, не картинка)

⚠️ **Не использовать скриншот** — он мелкий и нечитаемый. Рисовать интерфейс на чистом HTML+CSS как мокап. Минимальная структура:

- Тулбар браузера сверху (точки, адресная строка).
- Заголовок встречи: «Екатерина Иванова, вас пригласили на встречу!»
- Подзаголовок: «Рекрутер (удалённо) · 30 минут».
- Подсказка: «Нажмите на удобное время».
- Мини-календарь (месяц) с выделенной активной датой синим.
- 4 колонки дней с пустыми тире (нет слотов) и одной активной колонкой с временными слотами (12:45, 13:05, 13:25, 13:45, 14:05).
- Подпись «Расписание создано на Слотто».

Подробный референс — в существующем лендинге `recruto.ru/Recruiter`.

## 9. Что брать с лендинга и из вакансии под конкретную позицию

Когда делаем лендинг для конкретной вакансии Рекруто, **под бренд-китом мы не подразумеваем, что Claude знает специфику роли**. Под каждую вакансию нужно отдельно:

1. **Ссылка на оригинал вакансии на hh.ru** — для кнопки в шапке и для извлечения списка задач.
2. **Акценты лендинга** (1–3 штуки) — для каждой вакансии разные.
3. **Текст с описанием задач** — дословно из вакансии.
4. **Если есть видео-задание для кандидата** (как для интервьюеров) — ссылка на видео и описание задания.

Brand kit заменяет вопросы про **цвета, шрифты, логотип, описание компании, цифры, корпоративный голос**. Всё остальное всё равно надо приложить отдельно под каждую вакансию.

## 10. Чеклист — что использовать на лендингах Рекруто

- [ ] Цвета строго из палитры выше (`#2567fc`, без других ярких)
- [ ] Шрифт Montserrat подключён
- [ ] Шапка тёмная, логотип Рекруто (data-URI) слева
- [ ] Подвал тёмный, копирайт с правильной формулировкой про Сколково
- [ ] Цифры компании (400 тыс.+ / 1000+ / 9 из 10) присутствуют где-то на странице
- [ ] **Главные преимущества (автоматизация + гибкость) есть на странице — это сквозной блок**
- [ ] **Инструменты-доказательства (CRM/мессенджер/Слотто/экспертиза) — только 2–3 шт., релевантные под акцент**
- [ ] Используется «рекрутмент», не «рекрутинг»
- [ ] Аккредитация и Сколково сформулированы как «проект в экосистеме», не «мы — аккредитованная компания»
- [ ] Нет запрещённых слов из раздела 7
- [ ] Слотто (если используется) — мокап на CSS, не картинка
